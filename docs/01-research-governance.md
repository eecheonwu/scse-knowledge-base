# Research Governance

## Overview

The **SSOT-Centric Software Engineering (SCSE) Research Programme** is an ongoing research initiative focused on establishing a knowledge-centric approach to software engineering for the AI era. The programme explores how a **Single Source of Truth (SSOT)** can serve as the authoritative engineering knowledge system throughout the software lifecycle, enabling intelligent software development, explainable AI-assisted engineering, and continuous software evolution.

This repository serves as the central knowledge base for the research programme. It documents the research direction, literature, emerging concepts, ongoing work, publications, prototypes, and future roadmap. It complements the foundational SCSE framework and related research artifacts rather than replacing them.

---

# Vision

To establish **SSOT-Centric Software Engineering (SCSE)** as a practical and scientifically grounded software engineering paradigm that enables intelligent, traceable, and continuously evolvable software systems through the effective preservation and management of engineering knowledge.

---

# Mission

The mission of this research programme is to:

- Develop the theoretical foundations of SSOT-Centric Software Engineering.
- Advance methodologies for SSOT-Driven software development and software evolution.
- Investigate how AI-native software engineering can leverage structured engineering knowledge.
- Build practical frameworks, metrics, and open-source tools that demonstrate the applicability of SCSE.
- Validate proposed concepts through empirical studies, real-world case studies, and software prototypes.
- Contribute to the software engineering community through publications, technical reports, and open research.

# Research Focus Areas

The research programme currently focuses on the following areas:

- SSOT-Centric Software Engineering (SCSE)
- SSOT-Driven Software Engineering (SDSE)
- Architecture Memory
- Software Architecture Knowledge Management
- Architecture Knowledge Graphs
- Engineering Knowledge Ontology
- Knowledge Graph Engineering for Software Systems
- AI-Native Software Engineering
- Agentic Software Engineering
- Intelligent Implementation Planning
- Intelligent Task Planning
- SSOT Translation Quality Evaluation (STQE)
- SSOT-Driven Change Impact Analysis
- SSOT-Driven Feature Change and Addition
- Continuous Architecture Governance
- Software Evolution and Maintainability

These research areas collectively contribute to the development of a unified knowledge-centric software engineering framework capable of supporting both human engineers and AI software agents.

The programme is currently focused on:

- Refining the SCSE conceptual framework.
- Developing the SCSE research monograph.
- Building a canonical research library to support SCSE.
- Formalizing the SCSE Body of Knowledge.
- Designing quantitative metrics for evaluating knowledge preservation and software evolution.
- Developing AI-native software engineering workflows.
- Designing implementation planning and task planning methodologies driven by an engineering SSOT.
- Exploring knowledge graphs and ontologies as foundational representations of software engineering knowledge.

# Guiding Principles

The SCSE research programme is guided by the following principles:

- Engineering knowledge should be preserved as carefully as source code.
- Every significant engineering decision should be traceable.
- Software evolution should preserve architectural intent.
- AI systems should operate using authoritative engineering knowledge rather than fragmented documentation.
- Software engineering should continuously maintain semantic consistency across all engineering artifacts.
- Research should remain transparent, reproducible, and evidence-driven.

# Research Roadmap

The research programme is expected to evolve through five major phases.

## Phase 1 — Foundation

- Establish the theoretical foundations of SCSE.
- Develop the research knowledge base.
- Curate the canonical research library.

## Phase 2 — Framework Development

- Formalize the SCSE Body of Knowledge.
- Define core concepts, methodologies, and engineering metrics.
- Develop knowledge-centric software engineering workflows.

## Phase 3 — Prototype Development

- Build proof-of-concept implementations.
- Develop AI engineering tools and reusable software engineering agents.
- Validate workflows using real-world case studies.

## Phase 4 — Empirical Validation

- Conduct empirical software engineering studies.
- Evaluate proposed metrics and methodologies.
- Compare SCSE with existing software engineering approaches.

## Phase 5 — Dissemination

- Publish the SCSE research monograph.
- Publish peer-reviewed journal and conference papers.
- Release open-source software, datasets, and supporting resources.
